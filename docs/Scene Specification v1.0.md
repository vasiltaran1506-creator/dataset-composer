# Scene Specification v1.0
---
# 1. Introduction

Scene Specification определяет архитектуру Dataset Composer.

Dataset Composer не является генератором промптов.

Основной сущностью системы является Scene.

Промпт рассматривается исключительно как текстовое представление сцены.

Все внутренние процессы системы работают только с объектами Scene.

Библиотека промптов используется исключительно как источник текстовых описаний и не содержит никакой логики.

Вся логика построения сцен реализуется Scene Engine.

---

# 2. Goals

Основная цель проекта — генерация логически согласованных сцен для создания высококачественных датасетов обучения LoRA.

Система должна:

- создавать максимально разнообразные сцены;
- избегать логических противоречий;
- обеспечивать равномерное покрытие признаков датасета;
- быть независимой от конкретной модели генерации изображений;
- поддерживать различные форматы экспорта.

---

# 3. Design Principles

- Scene is the only source of truth.
- Prompt Library contains text only.
- Rules never contain prompt text.
- Every Scene must be logically valid.
- Coverage is more important than randomness.

# 4. Glossary

## 4.1 Core Concepts

### Scene
**Scene — атомарное описание одного изображения.**
- Scene содержит всю информацию, необходимую для генерации одного изображения или одного обучающего примера.
- Scene является главной сущностью Dataset Composer.
- Все остальные объекты существуют только как компоненты Scene.

### Scene Engine
**Scene Engine — модуль, создающий объекты Scene на основании библиотеки, ограничений и правил покрытия.**
- Он никогда не работает напрямую с текстовыми промптами.

### Prompt
**Prompt — текстовое представление Scene, предназначенное для использования конкретной моделью генерации изображений.**
- Prompt не является основной сущностью проекта.
- Он является результатом экспорта Scene.

## 4.2 Scene Concepts

### Character
**Character определяет неизменяемые свойства персонажа:**
- Цвет волос
- Цвет глаз
- Строение лица
- Телосложение
- Возраст 
- Пол

**Character не сожержит:**
- Поза
- Одежда
- Эмоции
- Окружение

### Outfit
**Outfit описывает одежду персонажа.**
**Outfit полностью независим от Character.**

### Pose
**Pose описывает положение тела персонажа.**
**Pose отвечат только за положние тела.**
**Pose не отвечает за действия.**

### Action 
**Action описывает действия персонажа.**
**Action не определяет положение тела.**

### Expression 
**Expression определяет выражение лица и направление взгляда персонажа.**

### Environment
**Expression описывает место.**
**Environment не описывает предметы.**

### Props
**Props описывает предметы.**

### Lightning
**Lightning описывает условия освещения.**

### Camera
**Camera описывает точку наблюдения.**
**Camera не описывает композицию.**

### Composition
**Composition описывает композицию кадра, расположение объектов в кадре.**

## 4.3 Constrains

### Constraint 
**Constraint - правило, ограничивающее совместимость компонентов Scene.**
**Constraint не принимает решений, он только сообщает ограничения.**

### Compatibility
**Compatibility описывает допустимые сочетания.**

### Dependency 
**Dependency описывает обязательные зависимости.**

### Exclusion
**Exclusion описывает несовместимости.**

### Requirement
**Requirement описывает обязательные наличия.**

## 4.4 Coverage System

### Coverage
**Coverage - статистическая мера представленности определенной характеристики в датасете.**

### Target Coverage
**Target Coverage - желаемое количество.**

### Coverage Engine
**Coverage Engine - модуль, анализирующий уже сгенерированные сцены и определяющий, какие характеристики необходимо генерировать дальше для достижения целевого покрытия.**

## 4.5 Prompt Library

### Prompt Library
**Prompt Library - коллекция текстовых фрагментов.**
**Prompt Library не содержит логики.**

### Prompt Fragment
**Prompt Fragment - минимальная текстовая единица.**

### Category 
**Category - логическая группа Prompt Fragment.**

## 4.6 Export System 

### Exporter
**Exporter - модуль преобразования Scene в другой формат.**

### Export Template 
**Export Template - шаблон формирования промпта.**

## 4.7 Dataset

### Dataset
**Dataset - коллекция Scene. Не коллекция Prompt.**

### Dataset Profile
**Dataset Profile - описывает, каким должен быть датасет до начала генерации. Именно этот профиль испольщует Coverage Engine для принятия решений.**

## Other 

### Semantic Validity
**Semantic Validity - сцена, не содержащая логических противоречий.**

### Dataset Diversity
**Dataset Diversity - количественная оценка разнообразия датасета. Она учитывает не только число уникальных промптов, но и распределение по позам, действиям, одежде, локациям, освещению, ракурсам и другим компонентам.**

## 5. Override System (Система преопределений и контроль пользователя)

`Dataset Composer` построен на строгих правилах семантической валидности (`Scene Rules`) и ограничениях персонажа (`Character Profile`). Однако, как профессиональный инструмент для создания датасетов, система должна предоставлять пользователю (Режиссеру) абсолютный контроль над процессом генерации, позволяя намеренно нарушать базовые правила для решения специфических творческих задач без переписывания кода.

### 5.1. Иерархия приоритетов (Priority Hierarchy)
При генерации сцены движок применяет фильтры в следующем порядке (от базовых законов к личным приказам):
1. **Scene Rules (Законы мира):** Базовая семантическая совместимость (описана в `.toml` файлах).
2. **Character Profile (Законы персонажа):** Предпочтения и табу конкретного персонажа.
3. **Run Config / UI Overrides (Приказ Режиссера):** Пользовательские настройки для текущей сессии генерации. **Имеет наивысший приоритет и может отменять пункты 1 и 2.**

### 5.2. Механизмы переопределения (Override Mechanisms)
Пользователь может создать `Run Config` (конфигурационный файл сессии или пресет в будущем UI), который модифицирует поведение `Scene Builder` и `Validator` для конкретной партии изображений.

Ключевые флаги переопределения:
*   `ignore_location_blacklists: true` — снимает жесткие запреты (Hard Constraints) на одежду или реквизит для выбранной локации (например, разрешает сгенерировать нижнее белье в школьном классе для специфической серии кадров).
*   `force_tags` / `force_categories` — принудительно задает использование определенных тегов или категорий, игнорируя веса и вероятности.
*   `validator_mode: "permissive"` — переводит Валидатор из режима "Блокировка и откат (Backtracking)" в режим "Предупреждение (Warning)". Сцена с логическим противоречием не отбрасывается, а попадает в датасет, а в логи записывается `⚠️ Warning: Rule broken by user override`.
*   `disable_character_lock: true` — временно отключает фиксированные черты персонажа (полезно для создания альтернативных версий или AU-сеттингов).

### 5.3. Философия применения
Базовые `.toml` файлы (`Scene Rules`) остаются неизменными и описывают "физику мира по умолчанию". `Override System` позволяет создавать изолированные сессии (например, "Специальная съемка" или "Сюрреалистичный эксперимент") без риска загрязнения глобальных правил проекта. Статистика `Coverage Engine` продолжает честно учитывать такие сцены в общей матрице покрытия датасета.
