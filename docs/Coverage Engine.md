# Coverage Engine v1.0 
 
 ---

## 1. Обзор и философия

 **Coverage Engine (Движок Покрытия)** — это stateless (не хранящий состояние) модуль, управляемый данными на диске. Он предназначен для анализа, отслеживания и математической балансировки семантического распределения сгенерированных сцен. Он гарантирует, что датасеты для обучения Character LoRA поддерживают оптимальное покрытие по ключевым визуальным осям, предотвращая переобучение на специфичных позах, локациях или ракурсах.
 
 **Ключевая Философия**: "Диск — это единственный источник истины."
 Движок не хранит никаких персистентных состояний, счетчиков или баз данных. Каждый анализ выполняется путем сканирования реальных файлов .txt на диске. Это гарантирует идеальную синхронизацию с рабочим процессом ручного курирования (удаление, перемещение или архивирование файлов никогда не сломает систему).

 ---

## 2. Ключевая терминология

 **Матрица покрытия** - Многомерная карта частот, показывающая количество сцен по каждой категории в отслеживаемых измерениях.

 **Семантическое Измерение** - Отслеживаемая ось категорий: Location (Локация), Action (Действие), Weather (Погода), Camera (Камера), Outfit Style (Стиль одежды).

 **Дефицит** - Категория, текущий процент которой падает ниже целевого порога.

 **Переизбыток** - Категория, текущий процент которой превышает целевой порог.

 **Целевой Профиль** - Желаемый шаблон распределения (по умолчанию: равномерный; кастомный: задается через YAML).

 **Взвешенная Генерация** - Корректировка вероятностей, применяемая к SceneBuilder для приоритизации дефицитных категорий во время генерации.

 **Stateless Сканирование** - Анализ папок по требованию, который читает файлы заново каждый раз, без кэширования состояния.

 **Цикл Курирования** - Workflow пользователя: Генерация → Рендер Картинок → Курирование (оставить/удалить/переместить) → Анализ → Балансирующая Генерация.

## 3. Архитектура и поток данных

[CLI Команда Пользователя]
        ↓
[CoverageTracker.scan(путь_к_папке)]
        ↓ (Читает .txt файлы, парсит теги)
[Tag Matcher] → Сверяет теги с загруженными словарями scene_rules
        ↓
[Matrix Builder] → Агрегирует счетчики по каждому измерению
        ↓
[Balancer] → Сравнивает с Целевым Профилем → Рассчитывает веса дефицита
        ↓
[SceneBuilder] → Применяет веса к случайному выбору → Генерирует сбалансированные сцены
        ↓
[Exporter] → Сохраняет новые .txt файлы → Цикл повторяется

## 4. Структура матрицы покрытия

Движок возвращает стандартизированный Python-словарь (сериализуемый в JSON):

{
  "folder_path": "Excellent Prompts",
  "total_scenes": 142,
  "dimensions": {
    "location": {"bedroom": 45, "library": 30, "beach": 5, "street": 20},
    "action": {"reading": 50, "sleeping": 25, "walking": 15},
    "weather": {"sunny": 60, "rain": 40, "fog": 10, "night": 20},
    "camera": {"close_up": 90, "full_body": 12, "cowboy_shot": 25}
  },
  "percentages": { ... }, # Автоматически рассчитано для каждого измерения
  "status": {
    "deficits": ["location.beach", "camera.full_body"],
    "overflows": ["camera.close_up", "location.bedroom"]

  "unmatched_tags_count": 3, # Теги, не найденные ни в одном словаре правил
  "scan_timestamp": "2026-07-01T14:30:00"

## 5. Логика Парсинга и Метчинга тегов

 ### 5.1 Чтение файлов

 - Рекурсивно сканирует целевую папку на наличие *.txt файлов.
 - Читает каждый файл в UTF-8, разбивает содержимое по ",", удаляет пробелы.
 - Элегантно пропускает пустые файлы или битые строки (логирует warning, продолжает работу).

 ### 5.2 Классификация тегов

 - Использует предварительно загруженные словари правил из ConfigLoader:
available_locations, available_actions, available_weathers, available_cameras
 - Итерируется по тегам промпта. Первое совпадение в измерении побеждает.
 - Пример на основе реальных файлов: Для luna_school_classroom_0060.txt парсер найдет school_classroom (локация), reading (действие) и close_up (камера), проигнорировав 1girl, solo, white shirt и book.
 - Игнорирует fixed_traits и теги одежды для отслеживания покрытия (в рамках v1).
 - Нераспознанные теги считаются в unmatched_tags_count, но не ломают парсинг.

 ### 5.3 Оптимизация производительности

 - Словари правил конвертируются в set() для поиска за O(1).
 - Поддерживает флаг --fast-scan: читает только имена файлов, если они следуют конвенции char_location_XXXX.txt.
 - Читает файлы потоково, чтобы держать использование RAM низким (<50MB даже для 10к файлов).

## 6. Целевые профиоли и Алгоритмы балансировки

### 6.1 Профиль по умолчанию
 
 - Целевой % = 100 / N, где N = количество известных категорий в этом измерении.
 - Пример: 12 локаций → цель = 8.33% на каждую.
 
### 6.2 Кастомный Профиль (YAML)
 
 Пользователи могут создать run-configs/target_face_focus.yaml:

 coverage_targets:
  camera:
    close_up: 0.60
    cowboy_shot: 0.20
    full_body: 0.15
    pov: 0.05
  weather:
    sunny: 0.40
    rain: 0.30
    night: 0.30

### 6.3 Алгоритм расчета весов
 
 1. Для каждого измерения вычисляется current_pct = count / total_scenes.
 2. Вычисляется deficit_score = max(0, target_pct - current_pct).
 3. Все оценки дефицита нормализуются в распределение вероятностей (сумма = 1.0).
 4. Применяется минимальный "пол" веса (например, 0.02), чтобы предотвратить блокировки с нулевой вероятностью.
 5. weight_map передается в SceneBuilder.build_scene().

### 6.4 Взвешенная генерация
 
 - Заменяет random.choice(categories) на random.choices(categories, weights=dimension_weights, k=1)[0].
 - Если дефицита нет, возвращается к равномерному рандому.
 - Веса пересчитываются каждый батч (по умолчанию батч = 50 сцен).

## 7. CLI интерфейс и команды ("Команда" - "описание")
 
 - python main.py analyze <папка> - Сканирует папку, печатает ASCII-матрицу покрытия с флагами дефицита/переизбытка.
 - python main.py generate <N> --balance-from <папка> - Генерирует N сцен с весами, направленными на исправление дефицита из <папка>.
 - python main.py generate <N> --target-profile <file.yaml> - Использует кастомные цели распределения вместо равномерных.
 - python main.py analyze <папка> --json - Выводит сырую матрицу в JSON (для внешних инструментов/GUI).
 - python main.py analyze <папка> --dimensions location,camera - Отслеживает только указанные измерения.

## 8. Точки интеграции ("Модуль" - "зона ответственности")
 - coverage_tracker.py - Ядро сканера, парсер, строитель матрицы, балансировщик.
 - config_loader.py - Загрузка кастомных целевых профилей (*.yaml).
 - scene_builder.py - Принимает словарь generation_weights, применяет взвешенный рандом к выбору измерений.
 - exporter.py - Вызывает трекер перед генерацией, передает веса, обрабатывает циклы батчей.
 - main.py - Роутер CLI, оркестрирует workflows analyze и --balance-from.

# 9. Граничные случаи и обработка ошибок ("Сценарий" - "поведение")
 
 - Пустая папка - Возвращает нулевую матрицу, печатает warning, элегантно завершает работу.
 - Битый .txt - Пропускает файл, логирует [WARN] Пропущен битый промпт: file.txt, продолжает.
 - Неизвестные теги - Игнорируются для покрытия. Считаются в unmatched_tags_count.
 - Измерение отсутствует в промпте - Считается как unspecified (не указано) в матрице этого измерения.
 - Датасет > 10,000 файлов - Использует потоковое чтение + прогресс-бар. Предлагает --fast-scan.
 - Сумма Target profile ≠ 1.0 - Автоматически нормализует проценты, печатает warning.
 - Ноль сцен всего - Отключает балансировку, возвращается к равномерной генерации.
